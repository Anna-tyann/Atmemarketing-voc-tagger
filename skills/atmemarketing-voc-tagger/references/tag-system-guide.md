# 标签体系构建完整指南

## 设计哲学

好的标签体系 = 洞察驱动 + 可度量 + 可比较 + 可行动

每个 L2 标签的提取结果，必须能回答以下问题：
1. 这个功能被多少用户关注？（声量）
2. 用户满意吗？（正面率）
3. 如果不满意，具体哪里不好？（原文引用）
4. 怎么改进？（可行动的建议）

---

## L1 类目设计

### 11 个通用 L1 类目（适用绝大多数实体产品）

| # | L1 名称 | 覆盖内容 | 示例 L2 |
|---|---------|----------|---------|
| 1 | **核心功能** | 产品最核心的使用效果 | 清洁效果、吸力、清洁能力 |
| 2 | **性能表现** | 动力/功率/速度/效率 | 水压、转速、加热速度 |
| 3 | **产品设计与做工** | 外观/材质/尺寸/配件 | 体积、重量、做工、材质 |
| 4 | **便携与出行** | 携带/移动/旅行场景 | 便携性、旅行适配、收纳 |
| 5 | **电池与续航** | 供电/充电/续航（充电产品） | 续航、充电速度、电池类型 |
| 6 | **易用性与操作** | 上手难度/交互/设置 | 易用性、安装、控制方式 |
| 7 | **价格与价值** | 性价比/价格感受 | 性价比、价格对比、值得度 |
| 8 | **可靠性与售后** | 故障/质保/客服/物流 | 可靠性、售后、包装、退货 |
| 9 | **品牌与推荐** | 品牌信任/推荐意愿 | 推荐度、品牌信任、配件 |
| 10 | **整体评价** | 总体满意/不满 | 整体满意、整体不满 |
| 11 | **其他** | 兜底 | 其他 |

### 根据品类增删 L1

- **有屏幕** → 加「屏幕显示」
- **有 App/软件** → 加「软件与连接」
- **食品/消耗品** → 加「口感与口味」「成分与安全」
- **服装** → 加「尺码与版型」「面料与舒适度」
- **美妆** → 加「肤感与吸收」「成分功效」
- **不插电产品** → 去掉「电池与续航」

## L2 标签设计

### 命名规范

- ✅ 使用用户语言：「清洁效果」而非「清洁效能指标」
- ✅ 具体可感知：「续航时间」而非「电池表现」
- ✅ 语义一致：同一 L2 下的表述要统一
- ❌ 避免技术术语：「水压强度与调节」而非「PSI 压力测量」
- ❌ 避免过于宽泛：「整体感觉」 → 应拆分到具体维度

### 关键词设计

每个 L2 需要 10-20 个英文关键词/短语：

```
好的关键词示例（"水箱容量"）:
["tank", "reservoir", "capacity", "hold water", "fill", "refill",
 "enough water", "not enough water", "small tank", "run out of water",
 "empty", "need more water", "halfway", "entire mouth", "full mouth",
 "canister", "water capacity"]

不好的关键词示例:
["water"]  — 太宽泛，会误匹配
["容量"]    — 中文关键词在英文评论中匹配不到
["好", "坏"] — 过于通用
```

### 关键词来源

1. 扫描 20-30 条该品类评论，提取高频名词和动词
2. 关注用户反复提到的功能和问题
3. 正负面词都要覆盖（情感分析会自动判断极性）
4. 多词短语优先于单词（"easy to use" > "easy"）

## 迭代优化

第一次打标后，检查以下指标并迭代：

1. **标签分布**：是否存在某个 L2 标签 0 匹配？→ 关键词太少或太生僻
2. **情感一致性**：5星评论是否产生了大量负面标签？→ 关键词中负面词占比过高
3. **L2 粒度**：某个 L2 标签提及次数超过总评论数 50%？→ 需要拆分为更细的 L2
4. **覆盖率**：大量评论只匹配了"整体评价"？→ 标签体系不够详细

每次迭代后重新打标，对比前后差异，直到标签体系稳定。
